### 查证结果
1. **台湾中石化跟冷能梯级利用无直接关系**：台湾中石化（台湾中国石油化学工业开发股份有限公司）在洋口港做的是化工仓储（丙烯腈、苯乙烯等液体化学品）+ 化工新材料生产（环己酮等），不是冷能用户

3. **管线桥不是冷能管道**：管线桥承载天然气管道和化工品管道，目前没有专门的冷媒输送管道。冷能利用主要在岛上（杭氧空分），陆地数据中心的冷能输送方案仍在"加快推进配套方案细化"阶段
4. **梯级利用实际链条**：深冷级（空分+乙烷分离）→ 中冷级（发电+制液氢）→ 浅冷级（数据中心散热+养殖+制冰）

## 修复：博文 404 问题（pubDatetime 时区）

### 问题
- 推送后博文 404，但构建成功（GitHub Actions 返回 success）
- 本地构建发现 `dist/posts/china-telecom-nantong-yangkou-port-datacenter/` 只有 `index.png`（OG图片），没有 `index.html`
- 根因：`pubDatetime: 2026-07-15T12:00:00` 没有时区标识，GitHub Actions 在 UTC 时区下解析为 UTC 12:00（= 北京时间 20:00），而构建运行时是 UTC ~05:00，`postFilter` 判定为"定时发布"跳过了 HTML 生成
- OG 图片生成器（index.png.ts）有自己的 getStaticPaths，不过滤定时发布，所以 index.png 正常生成

### 修复
- 给日期加 `+08:00` 时区标识：`pubDatetime: 2026-07-15T08:00:00+08:00`（= UTC 00:00，构建时已过去）
- commit 1a78ee8 已推送，构建成功，页面返回 200

### 教训
- 博文 frontmatter 的 pubDatetime **必须带时区标识**（如 `+08:00`），否则在 GitHub Actions UTC 环境下可能被判定为定时发布
- 之前 NVIDIA 博文没出问题是因为 `pubDatetime: 2026-07-14T23:20:00+08:00` 已经带了时区
